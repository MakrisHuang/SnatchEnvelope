Snatch Red Envelope

模擬線上多人搶紅包

這是一個模擬多人在線上，搶票或是搶商品的網站；本網站採用流行的SSM框架（Spring MVC + Spring + MyBatis）， 並使用Redis為暫存伺服器，來快速響應使用者需求，並也能在大量請求下快速地回傳正確結果。 本網站另採用Spring Boot + Spring Security驗證使用者身份，並設計錯誤頁面(點我查看)回應使用者錯誤的請求。

點我登入

本網站採用全註解方式配置Spring Boot，並架設於Amazon Web Service (AWS)上以利使用者瀏覽。 在前端頁面則是使用常用的Thymeleaf框架，不僅原生支援HTML 5，也能用MVC的方式進行渲染。另外， 為了開發dev版及production版本，亦採用@Profile來讀取外部properties以利切換資料庫來源。

Spring Boot

現今網站不僅架構複雜，同時也要面對大量請求的封包。而對於進行商品搶購或是搶紅包的場景，Redis能把 資料暫存以利提供高速的數據讀取，來減少資料庫的壓力。本網站的搶紅包因需要在短時間內響應上千以上的 請求，故使用Redis這NoSQL資料庫提供健全、有效的服務。

Redis.io

使用MyBatis作為資料庫存取框架的一大好處是MyBatis能讓開發者自訂SQL，因而能準確定義SQL規則， 以利優化性能，同時也能加速開發，對於性能要求高、靈活的系統十分有利。MyBatis的映射模式也能以 面向對象的方式讀取和寫入資料，非常方便。
